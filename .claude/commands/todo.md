# TODO.md 수정사항 처리

## 워크플로우

1. **CLAUDE.md 읽기** — 프로젝트 가이드라인, 디자인 시스템, 컨벤션 확인
2. **TODO.md 읽기** — 수정사항 목록 확인
3. **위에서부터 순서대로 처리** — 각 항목을 하나씩 구현
4. **항목마다 빌드 확인** — `npm run build` 성공 확인. 실패하면 즉시 수정
5. **완료된 항목 삭제** — 해당 항목을 TODO.md에서 삭제
6. **모든 항목 처리 후 냉철한 셀프 리뷰** — 아래 체크리스트 기준으로 검토
7. **리뷰 통과 시 커밋+푸시**

---

## 셀프 리뷰 체크리스트 (모든 항목 처리 후 반드시 수행)

### 코드 품질
- [ ] `npm run build` 에러 없음
- [ ] 사용하지 않는 import 제거
- [ ] console.log 남아있지 않음 (console.warn/error는 허용)
- [ ] 하드코딩된 값 없음 (상수/유틸 함수 사용)

### 디자인 시스템 준수
- [ ] CLAUDE.md의 디자인 토큰 사용 (primary: #059669, 폰트, 간격 등)
- [ ] Tailwind 클래스 일관성 (cn() 유틸 사용)
- [ ] 모바일 375px~430px 기준 레이아웃 확인
- [ ] Lucide 아이콘 통일 (다른 아이콘 라이브러리 혼용 금지)

### 기능 검증
- [ ] 구현한 기능이 TODO.md 요구사항을 정확히 충족
- [ ] 기존 기능 깨진 것 없음 (사이드 이펙트 확인)
- [ ] API 연동 필드명이 백엔드 응답과 일치
- [ ] 방어 코드 포함 (null/undefined 체크, optional chaining)

### 리뷰 실패 시
- 문제를 수정하고 다시 빌드 확인
- 리뷰 체크리스트를 다시 통과할 때까지 반복

## 커밋 규칙

리뷰 통과 후:
```bash
git add -A
git commit -m "feat/fix: 변경사항 요약 (TODO.md 항목 기반)"
git push origin main
```

커밋 메시지는 처리한 항목 내용을 기반으로 작성.
여러 항목이면 가장 핵심적인 변경을 메시지로.

---

## 주의사항
- TODO.md에 항목이 없으면 "처리할 수정사항이 없습니다" 로 종료
- 각 항목은 독립적으로 처리 (하나 실패해도 나머지 진행)
- 빌드 실패하면 해당 항목 롤백 후 다음 항목 진행
- TODO.md에서 완료된 항목만 삭제하고, `_(모든 항목 처리 완료)_`는 모든 항목 처리 후 추가
