# /implement — 계획에 따라 구현 실행

$ARGUMENTS 에 해당하는 작업을 **사전에 수립된 계획에 따라** 구현해.

## 전제 조건
- 반드시 /plan이 먼저 실행되어 PROGRESS.md에 해당 Phase의 계획이 존재해야 해.
- 계획이 없으면 "⚠️ 계획이 없습니다. /plan을 먼저 실행하세요." 라고 안내하고 중단해.

## 구현 순서

### 1단계: 계획 재확인
1. PROGRESS.md에서 해당 Phase의 계획을 읽어.
2. 계획의 **구현 항목**과 **디자인 체크포인트**를 확인해.
3. "🔨 [Phase N] 구현을 시작합니다. 계획 항목 N개." 라고 알려줘.

### 2단계: 순차 구현
- 계획의 구현 항목을 **1번부터 순서대로** 진행해.
- 각 항목 완료 시 PROGRESS.md의 디자인 체크포인트를 중간 체크해.
- 계획에 적힌 공통 컴포넌트를 재사용하고, 계획에 없는 새 파일은 만들지 마.

### 3단계: 디자인 시스템 준수 확인
구현하면서 다음을 지속적으로 확인해:
- Tailwind 클래스만 사용 (인라인 스타일 금지)
- 디자인 토큰은 tailwind.config.js에 정의된 것만 사용
- 터치 타겟 44px 이상 유지
- input font-size 16px 이상 (iOS 줌 방지)
- 페이지 하단 pb-24 (BottomNav 가림 방지)
- 빈 상태 / 로딩 상태 / 에러 상태 UI 포함
- Primary 컬러 = 에메랄드 그린(bg-primary, text-primary)

### 4단계: 중간 점검
- 모든 항목 구현 후, 계획의 디자인 체크포인트를 하나씩 대조해.
- 미충족 항목이 있으면 즉시 수정해.
- "✅ 구현 완료. /verify로 빌드 검증을 진행하세요." 라고 알려줘.

## 절대 규칙
- 계획에 없는 것을 임의로 추가하지 마.
- 기존 코드를 계획에 명시되지 않은 방식으로 수정하지 마.
- 확신이 없으면 멈추고 물어봐.
