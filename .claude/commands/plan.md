# /plan — 작업 전 상세 계획 수립

$ARGUMENTS 에 해당하는 작업의 상세 계획을 수립해.

## 작업 순서

### 1단계: 현황 파악
1. CLAUDE.md를 읽고 디자인 시스템, 모바일 퍼스트 원칙, 코딩 컨벤션을 재확인해.
2. PROGRESS.md를 읽고 현재까지 완료된 작업과 다음 작업을 확인해.
3. 기존 코드를 훑어보고 현재 프로젝트 상태를 파악해 (어떤 컴포넌트가 있고, 어떤 패턴이 사용되고 있는지).

### 2단계: 계획 작성
PROGRESS.md의 해당 Phase 아래에 다음 형식으로 **상세 계획**을 추가해:

```markdown
#### 📝 [Phase N] 계획 (작성 시각: YYYY-MM-DD HH:MM)

**목표**: 이 Phase에서 달성할 것을 한 줄로 요약

**구현 항목**:
1. [파일명] — 구현할 핵심 내용 설명
2. [파일명] — 구현할 핵심 내용 설명
3. ...

**사용할 공통 컴포넌트**: (기존 것 중 재사용할 것)
**새로 만들 컴포넌트/훅**: (필요시)

**API 연동**:
- `GET/POST/PATCH/DELETE 엔드포인트` → 용도

**디자인 체크포인트**:
- [ ] 모바일 375px 기준 레이아웃
- [ ] Primary 에메랄드 그린(#059669) 사용
- [ ] 터치 타겟 44px 이상
- [ ] 빈 상태 / 로딩 상태 / 에러 상태 처리
- [ ] 페이지별 고유 체크포인트들...

**예상 커밋 메시지**: `feat: ...`

**위험 요소 / 주의사항**: (있으면)
```

### 3단계: 계획 확정
- 계획을 작성한 후 "📋 계획 수립 완료. 검토 후 /implement로 진행하세요." 라고 알려줘.
- 사용자가 수정 요청을 하면 계획을 업데이트해.

## 원칙
- 계획은 반드시 PROGRESS.md에 기록해 (휘발되지 않게).
- 구체적으로 작성해. "페이지를 만든다" 수준이 아니라 "어떤 컴포넌트를 어떻게 배치하고, 어떤 API를 호출하고, 어떤 상태를 관리할지"까지.
- 계획에 없는 작업은 구현 단계에서 하지 않아. 추가 필요 시 계획을 먼저 수정해.
