# /next — 다음 작업을 계획 기반 사이클로 진행

PROGRESS.md를 확인하고 다음 미완료 Phase를 **계획 → 구현 → 검증 → 리뷰 → 커밋** 사이클로 진행해.

## 워크플로우

```
┌─────────────────────────────────────────────────┐
│  1. /plan    → 계획 수립 + PROGRESS.md에 기록    │
│  2. /implement → 계획에 따라 구현                │
│  3. /verify  → 빌드 + 기술 점검                  │
│  4. /review  → 최초 계획 & 가이드라인 냉철 대조   │
│       ├─ ✅ 통과 → 5단계로                      │
│       └─ ❌ 실패 → 수정 후 3단계 재실행          │
│  5. /commit  → 커밋 + 푸시 + PROGRESS 업데이트   │
└─────────────────────────────────────────────────┘
```

## 실행 순서

### 1단계: 다음 작업 식별
- PROGRESS.md에서 첫 번째 미완료(- [ ]) Phase를 찾아.
- 해당 Phase의 내용을 확인해.

### 2단계: 계획 수립
- /plan 워크플로우를 실행해.
- PROGRESS.md에 상세 계획이 기록되었는지 확인해.

### 3단계: 구현
- /implement 워크플로우를 실행해.
- 계획의 항목을 하나씩 순서대로 구현해.

### 4단계: 검증
- /verify 워크플로우를 실행해.
- 빌드 실패 시 수정, 최대 3회.

### 5단계: 최종 리뷰
- /review 워크플로우를 실행해.
- ❌ 판정 시: 문제 수정 → /verify → /review 재실행.
- 리뷰 반복은 최대 2회. 2회 실패 시 멈추고 보고.

### 6단계: 커밋 & 푸시
- /commit 워크플로우를 실행해.
- PROGRESS.md 업데이트 확인.

### 7단계: 다음 안내
- 완료 후 "✅ Phase N 완료. 다음: Phase N+1. /next로 계속 진행 가능." 라고 알려줘.

## 규칙
- 한 번에 하나의 Phase만 진행해.
- 각 단계를 건너뛰지 마. 특히 /plan과 /review는 절대 생략 금지.
- 에러 발생 시 해당 단계에서 멈추고 사용자에게 보고해.
